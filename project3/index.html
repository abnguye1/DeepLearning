Objective

This project explores three core ideas behind modern sequence modeling: basic attention, multi-head attention, and the Transformer architecture. The central goal is to understand not only how to implement these models, but why they improve machine translation compared with traditional RNN encoder-decoder baselines.

Across all experiments, I evaluate translation quality with BLEU score and monitor optimization behavior through training and validation loss curves. The report focuses on architectural decisions, hyperparameter effects, interpretability via attention visualization, and practical trade-offs.

How Attention Works (Conceptual Introduction)

Basic Attention in Encoder-Decoder Models

In a vanilla Seq2Seq model, the decoder relies heavily on a single fixed context vector from the encoder, which can become a bottleneck for long or complex sentences. Attention addresses this by letting the decoder compute a weighted combination of all encoder hidden states at each decoding step. These weights indicate which source positions are most relevant to the current target token prediction.

Multi-Head Attention

Multi-head attention extends this idea by projecting queries, keys, and values into multiple subspaces and attending in parallel. Instead of one monolithic alignment, each head can specialize in different linguistic patterns, such as local word alignment, long-range dependencies, or phrase structure. Head outputs are concatenated and linearly transformed, allowing the model to integrate diverse contextual signals.

Transformer Architecture

The Transformer removes recurrence and relies entirely on attention and feed-forward blocks. Positional encodings provide token order information, while stacked self-attention layers model token interactions globally. Compared to RNNs, Transformers train more efficiently in parallel and often capture long-distance dependencies more effectively, which usually translates into stronger BLEU performance.
